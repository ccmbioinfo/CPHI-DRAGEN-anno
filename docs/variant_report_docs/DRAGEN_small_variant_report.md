## Summary

This document describes the CSV variant report generated by <https://github.com/ccmbioinfo/CPHI-DRAGEN-anno/tree/main/workflow/scripts/cre>, originally written by [Sergey Naumenko](https://github.com/ccmbioinfo/cre/tree/master). The report is based on GEMINI (<https://gemini.readthedocs.io/en/latest/>), VEP, gnomAD population allele frequencies, dbNSFP, and vcfanno annotations.

## 1\. Filters

### 1.1 Filters of variant calling programs

We filter out non-PASS variants from the DRAGEN VCF.

### 1.2 Filters applied after GEMINI annotation and before report generation

We report small variants (SNVs and indels) using the GRCh38 reference.

**wgs.coding:** broadly, medium to high impact non-synonymous variants.

  - gnomad\_fafmax\_faf95\_max \< 1% (or absent) - rare.
  - if the variant has a clinvar annotation, alt\_depth \>=1 in any sample
  - otherwise, alt\_depth \>= 3 in any sample and impact\_severity does not equal LOW
  - if gnomad\_fafmax\_faf95\_max \>1% and alt\_depth \>=1 in any sample and clinvar status is pathogenic or likely pathogenic and clinvar status is not ‘no assertion provided’

See also Ensembl VEP classification of variant impacts: <https://useast.ensembl.org/info/genome/variation/prediction/predicted_data.html>

**wgs.panel:** variants of any impact, i.e. including synonymous, intronic, UTR variants, but restricted to a set of genes defined by patient-specific HPO-terms.

  - gnomad\_fafmax\_faf95\_max \< 1% (or absent) - rare
  - if the variant has a clinvar annotation, alt\_depth \>=1 in any sample
  - otherwise, alt\_depth \>= 3 in any sample
  - if gnomad\_fafmax\_faf95\_max \>1% and alt\_depth \>=1 in any sample and clinvar status is pathogenic or likely pathogenic and clinvar status is not ‘no assertion provided’

**wgs.panel-flank:** same as wgs.panel, but includes variants in 100kb flank of HPO-associated genes

**wgs.denovo:** de novo variants (i.e. DRAGEN-called denovos, `FORMAT/DN = DeNovo`) of any impact

**wgs.high.impact:** ultra-rare variants in all genes, filtered to include only variants predicted to be deleterious/impactful.

  - gnomad\_fafmax\_faf95\_max \< 0.1% (or absent) - ultra-rare
  - if the variant has a clinvar annotation, alt\_depth \>=1 in any sample
  - otherwise, alt\_depth \>= 3 in any sample
  - CADD \>=10 or CADD is missing (rare indels) or SpliceAI \>= 0.2 or abs(promoterAI) \>= 0.1
  - Intergenic variants are filtered out

Non-coding prediction scores thresholds above which we consider a variant to be impactful:

  - CADD: 10
  - ncER: 95.95
  - ReMM: 0.9585
  - LINSIGHT: 0.9828

Thresholds taken from table 3\* in <https://doi.org/10.1016/j.gpb.2022.02.002>, with the exception of CADD; the study used v1.3, and we use v1.7. We chose the CADD threshold based on [this paper](https://doi.org/10.1093/nargab/lqaf157).

### 1.3 1000 Genomes allele frequencies and counts

The 1000 Genomes sequence variant database (n=3,202, 602 trios) used to annotate variants was created from [DRAGEN SV callsets v4.4.7](https://registry.opendata.aws/ilmn-dragen-1kgp/) using the following methods:

  - Download per-sample DRAGEN SV VCFs from the public 1000 Genomes DRAGEN S3 bucket
  - Run `bcftools norm` and `vt decompose` to left-normalize and decompose multiallelic variants. 
  - Retain only `FILTER=PASS` variants and remove all format fields
  - Merge single-sample VCFs into one cohort-level VCF
  - Use `bcftools +fill-tags` to populate cohort INFO tags: `AC, AC_Hom, AC_Het, AN, AF`
  - Recalculate `AF` from `AC` using a fixed cohort size of 3102 samples (`AN = 6204`).
  - Derive `NHOMALT` from `AC_Hom / 2`.
